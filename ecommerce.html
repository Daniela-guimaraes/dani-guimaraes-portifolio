Visão Geral

Implementação e deploy robusto de uma API de E-commerce utilizando o framework Flask e persistência de dados com SQLite. O foco principal deste projeto foi a entrega contínua (CI/CD) e a depuração de problemas complexos de infraestrutura na AWS Elastic Beanstalk (EB).

O projeto superou falhas críticas de inicialização do servidor (Gunicorn) e do banco de dados (SQLite), garantindo que a aplicação fosse executada de forma estável em um ambiente de produção Linux (Amazon Linux 2023).

Principais Aprendizados e Habilidades:

  • Resolução de Erros de Servidor (502 Bad Gateway): Configuração correta do `Procfile` e Gunicorn para atender ao Nginx na porta padrão do EB.
  • Depuração de Contexto de Aplicação (500 Internal Server Error): Solução do erro fatal de DB ausente, implementando a criação do banco de dados SQLite de forma automática e resiliente via *hook* `before_request` do Flask.
  • SSH e Isolamento de Ambiente: Utilização avançada de `eb ssh` e comandos Linux (`find`) para depurar problemas de `PATH` e ambiente virtual (`venv`) da AWS.

Resultado do teste POST no Postman, confirmando que a aplicação está totalmente funcional na AWS. O Status 200 OK e a mensagem de sucesso provam que o Gunicorn está ativo e o banco de dados (SQLite) foi criado e está persistindo dados corretamente.

⚠️ Depuração de Erro 500 Interno

Este print documenta o erro crítico que indicava que o servidor estava rodando, mas a lógica da aplicação falhava internamente. A causa foi diagnosticada como a ausência do arquivo SQLite (`ecommerce.db`), que precisava ser criado no ambiente de produção.

Monitoramento de Integridade na AWS

Visão geral no console do AWS Elastic Beanstalk (EB). O status Ok e a versão de plataforma Python 3.12 confirmam que a aplicação foi implantada e está em estado saudável após a correção dos erros de Gunicorn e a implementação da criação automática do DB.

Depuração de Infraestrutura e venv da AWS

Registro de um dos momentos de depuração via SSH. O erro flask: command not found e a subsequente falha de sintaxe de comandos Python complexos demonstram a necessidade de habilidades avançadas de troubleshooting em ambientes de produção Linux para contornar limitações do ambiente virtual (venv) da AWS.
